# 项目配置指南

本文档详细说明如何配置项目审核规则的CSV文件。

## 快速开始

1. 复制示例文件：
```bash
cp ref.example.csv ref.csv
```

2. 根据您的项目需求编辑 `ref.csv` 文件

3. 确保在 `config.json` 中正确配置了项目配置路径：
```json
{
  "project_config_path": "ref.csv",
  "enable_project_audit": true
}
```

## CSV文件格式

### 必需字段

| 字段名 | 说明 | 示例 |
|--------|------|------|
| 项目名称 | 项目的唯一标识符，用于匹配稿件类别 | `南京大牌档` |
| 话题标签 | 稿件必须包含的话题标签，用空格分隔 | `#美团黑钻会员 #南京大牌档` |
| 利益点标准 | 稿件必须准确描述的利益点和权益内容 | `黑钻直升「状元」；绑定即领满100-50进店见面礼` |

### 可选字段

| 字段名 | 说明 | 示例 | 默认值 |
|--------|------|------|--------|
| 项目介绍 | 项目的详细描述（仅供参考，不参与审核） | `南京特色餐厅品牌合作` | - |
| 口令要求 | 稿件中必须包含的优惠码或活动口令 | `NJDPD2024` | 无要求 |
| 审核严格度 | 审核的严格程度：`strict`/`normal`/`loose` | `strict` | `normal` |

## 字段详细说明

### 1. 项目名称

**作用**：用于匹配飞书表格中的"权益类型"字段，确定使用哪个项目的审核规则。

**匹配规则**：
- 精确匹配优先：`南京大牌档` 完全匹配
- 包含匹配：`南京` 可以匹配 `南京大牌档`
- 反向匹配：`南京大牌档限时活动` 可以匹配 `南京大牌档`

**注意事项**：
- 必须唯一，避免重复
- 建议使用简洁明确的名称
- 避免使用特殊字符

### 2. 话题标签

**作用**：检查稿件是否包含所有必需的话题标签。

**格式要求**：
- 多个标签用空格分隔
- 每个标签建议以 `#` 开头
- 支持中文和英文标签

**审核逻辑**：
- LLM会智能识别同义词和变体
- 例如：`#美团黑钻会员` 和 `#黑钻会员` 被认为是相同的
- 支持简写和全写的匹配

**示例**：
```csv
#美团黑钻会员 #南京大牌档 #联合会员
```

### 3. 利益点标准

**作用**：确保稿件准确描述项目的核心利益点和权益内容。

**格式建议**：
- 多个利益点用分号（；）分隔
- 描述要具体明确，包含数字、比例等关键信息
- 避免模糊或夸大的表述

**审核要点**：
- 检查数字准确性（折扣、次数、金额等）
- 验证权益描述的完整性
- 识别虚假或夸大宣传

**示例**：
```csv
黑钻直升「状元」；绑定即领满100-50进店见面礼；免排队（节假日不可用）
```

### 4. 口令要求

**作用**：验证稿件中包含的优惠码、活动口令等是否正确。

**审核特点**：
- 精确匹配，大小写敏感
- 不允许拼写错误或变体
- 如果字段为空，则跳过口令检查

**常见口令格式**：
- 品牌缩写 + 日期：`NJDPD2024`
- 活动名称缩写：`VALORANT400`
- 特殊代码：`STRICT123`

### 5. 审核严格度

**可选值**：
- `strict`：严格模式，要求完全匹配，不允许任何变体
- `normal`：标准模式，允许合理的同义词和表达变体
- `loose`：宽松模式，相对灵活的匹配规则

**影响范围**：
- 话题标签的匹配严格度
- 利益点描述的验证标准
- 口令检查的容错程度

## 配置最佳实践

```csv
项目名称,说明
南京大牌档,✅ 简洁明确
酒店权益,✅ 功能导向
美团黑钻会员南京大牌档联合会员权益活动,❌ 过长复杂
项目1,❌ 无意义命名
```

### 2. 话题标签设置

```csv
话题标签,说明
#美团黑钻会员 #南京大牌档,✅ 核心标签齐全
#美团 #餐厅,❌ 过于宽泛
#美团黑钻会员 #南京大牌档 #联合会员 #黑钻许愿真有用 #美食,❌ 标签过多
```

### 3. 利益点描述

```csv
利益点标准,说明
黑钻直升「状元」；满100-50优惠,✅ 具体明确
超值优惠；会员专享,❌ 模糊不具体
免费升房；免费早餐；85折住宿,✅ 具体权益清单
```

### 4. 审核严格度选择

| 项目类型 | 建议严格度 | 原因 |
|----------|-----------|------|
| 金融理财 | `strict` | 涉及金额，需要精确 |
| 品牌合作 | `normal` | 平衡准确性和灵活性 |
| 娱乐活动 | `loose` | 创意表达，相对灵活 |

## 常见问题

### Q1: 如何处理项目名称匹配不上的情况？

**A**: 检查以下几点：
- 确保CSV中的项目名称与飞书表格中的"权益类型"字段匹配
- 可以使用部分匹配，如用"南京"匹配"南京大牌档"
- 检查是否有拼写错误或多余的空格

### Q2: 话题标签检查失败怎么办？

**A**: 可能的原因：
- 稿件中缺少必需的标签
- 标签格式不正确（缺少#符号）
- 标签顺序或大小写问题
- 可以适当调整审核严格度

### Q3: 利益点检查总是失败？

**A**: 检查要点：
- 确保利益点描述具体明确
- 避免使用模糊词汇
- 数字和百分比要准确
- 可以先用`loose`模式测试

### Q4: 如何测试配置是否正确？

**A**: 建议步骤：
1. 先创建一个简单的测试项目
2. 使用较少的标签和利益点
3. 选择`normal`审核模式
4. 逐步增加复杂度

## 技术说明

### CSV编码
- 文件编码：UTF-8
- 分隔符：英文逗号（,）
- 换行符：支持Windows（\r\n）和Unix（\n）格式

### 特殊字符处理
- 包含逗号的内容需要用双引号包围
- 双引号内的双引号需要用两个双引号转义
- 建议避免使用特殊字符，简化配置

### 性能考虑
- CSV文件大小建议控制在1MB以内
- 项目数量建议不超过1000个
- 系统会自动缓存配置，提高性能
